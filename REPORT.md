[고려사항]
0. 동시에, 어떤 페이지에서도 재사용 가능하도록 범용성을 갖추는 것을 목표로 할 것, 즉 공통으로 쓰는 코드는 수정이 되면 안됨. 
1. 등록 화면이 팝업으로 되어있을 수도 있고, 아닐 수 도 있음
2. 아이디 중복 확인 버튼 이름이 중복 확인,중복확인, 중복체크, 중복 체크 이런식으로 차이가 있을 수 있음
 -> 이름이 다르고 뛰어쓰기 여부 차이가 있을 수 있음
3. 화면에 있는 인풋타임, 버튼 타임, 드롭박스, 라디오 박스 등 다양한 타임이 존재하는데, 어느 화면에는 라디오버튼, 드롭박스 등등 없는 경우가 있을 수 있음
-> 테스트 케이스에 있는 요소를 확인 한,  요소에 대한 타입을 확인해줘야함. 
4. 카탈론스튜디오 테스트 케이스 부분, 요소추출 은 사전에 다 진행할 예정 다만 버튼 클릭 과 같은 건 추출하지 않음, 코드 내에서 버튼 클릭은 자동으로 해줘야함. 
5. 아이디 검증 시, 중복체크 버튼 클릭으로 진행하며, 아이디 부분 검증이 마지막으로으로 할때 중복체크 버튼이 눌러져있어야함. 
6. 아이디 검증 제외하고 등록버튼을 매번 클릭 후 확인해야함. 
7. 등록 버튼 클릭 -> 등록을 하시겠습니까 (팝업) -> 등록, 취소,(팝업) -> 팝업 안에 등록버튼을 클릭했을때 등록이 완료 되었습니다. (팝업) 이런식으로 뜰 수 있어서, 테스트 케이스 요소추출 한 거 마지막 으로 돌아가서 다음 테스트 진행해야함
8. 실패에 대한 케이스 완료 후, 마지막 동작은 등록 성공이 나와야함
9. 실패에 대한 케이스 일 경우 
10. 모든 필드가 테스트 시작 전 상태로 복원야함. 
11. Katalon 리포트에 상세한 박스 형태 로그 출력    
12. 실패 시 스크린샷 자동 저장 (Reports 폴더)
13. 실패 시 빨간색으로 떠야하며 주황색으로 뜨면 안되고,  레포트에서는 Failed 에 출력되어야함,
14. 코드는 최대한 간단하게, 누가바도 알아보기 쉽게, 
15. 성공 실패 기준은, 실패할 해야할 케이스를 입력을 햇을때, 사용가능한... / 등록되었습니다. 등록 성공하였습니다. 완료되었습니다... 등등 이런 예외 케이스를 했을때, "성공" 이라는 단어 의미가 있을때 실패라고 해주고 정상를 햇을때해는 성공에 대한 메시지가 아닌 실패에 대한 메시지가 나오면 실패야. 
16. 화면에 초기화 버튼이 있다면, 초기화 후 등록 버튼을 클릭해줘야함.
